A. 比較対象モデル名の取り違え

原稿では頻出していた誤記の筆頭カテゴリ。

❌ A-1：Claude Opus 4.6 は公式比較対象ではない

誤：原稿は「フロンティア級の対抗馬として Claude Opus 4.6」を筆頭に並べていた。また第13章・17章・エピローグで SWE-bench比較・価格比較・捏造率比較の主要カウンターパートとして登場。

正：Qwen3.6-Plus 公式比較表で並べられているフロンティア4モデルは Claude Opus 4.5 / GPT-5.2 / Gemini-3 Pro / Kimi-K2.5。 Opus 4.6 や GPT-5.4 は発表時点で既に存在していたが、公式ベンチマーク表の主要比較軸からは外されていた点を、むしろコミュニティから指摘されていた経緯がある。筆頭モデル名として不正確。

❌ A-2：RealWorldQA の比較相手は Sonnet 4.5 ではなく Gemma4-31B

誤：「35B-A3B の RealWorldQA で クローズドモデルである Claude Sonnet 4.5 の 70.3点を上回る」。

正：公式 35B-A3B 比較表で 70.3 と対比されているのは Gemma4-31B（オープンウェイト）。対象モデルの取り違え。

❌ A-3：Plus 側 RealWorldQA の比較相手

誤：原稿は Plus 側の RealWorldQA でも「Claude Sonnet 4.5」を比較相手に置いていた可能性あり。

正：公式ブログで RealWorldQA の比較相手として掲載されているのは Claude 4.5 Opus（Sonnet ではない）／スコア 77.0。 「Claude Sonnet 4.5 = 70.3」という具体値の裏付けは公式資料・第三者ベンチのいずれにも見当たらない。

❌ A-4：安全性スコアの帰属

誤：原稿は「GPT-5.4（87.3）」「Claude（87.2）」を本体モデルのスコアとして扱った。

正：BridgeBench の Security Bench で 87.3 を記録しているのは GPT-5.4 Mini（ミニ版）であり、GPT-5.4 本体ではない。 Claude の 87.2 は Sonnet 4.5 のスコアであり、Opus の値ではない。Opus 4.6 の BridgeBench Security Bench 公開スコアは別数値（公表ベースでは 90 台半ば）。

B. ベンチマーク数値の誤り

❌ B-1：SWE-bench Pro「大きく上回る」は誤り

誤：「Qwen3.6-Plus の 56.6点 が Claude Opus 4.5 の 45.9点 を大きく上回る」。

正：Claude Opus 4.5 の SWE-bench Pro スコアは Qwen公式ブログ表で 59.3、TokenCost集計で 57.1（57点前後）。 Qwen3.6-Plus の 56.6点は Opus 4.5 を「大きく上回る」のではなく、むしろやや下回る水準。

❌ B-2：SWE-bench Verified の Claude側数値

誤：「Claude Opus 4.6 80.8点」を公式比較対象として扱う。

正：公式比較表に載っているのは Claude Opus 4.5 の 80.9点。 Opus 4.6 の 80.8 は第三者サイト（digitalapplied 等）が別途集計したもの。公式ベンチの比較相手は 4.5 が正。

❌ B-3：AIME の年次表記

誤：「AIME 2026（数学オリンピック予選）で 95.3点」。

正：Qwen3.6-Plus スコア 95.3 に対応する公式表記は AIME25（2025年のAIME、スコア95.3）。 AIME 2026 のスコアは、オープンウェイト版 35B-A3B の community-submitted evaluation 側で別途 92.7 として公表されているもの。原稿は Plus のスコアを「AIME 2026」に誤って紐付けている。

❌ B-4：HMMT の表記

誤：「HMMT Feb 25」という略記。

正：HMMT の 96.7 は HMMT 2025（2月開催分）を指すので、スコア自体は正だが、「Feb 25」表記は年号略記として誤解を招く（2月25日と読める）。本サイトでは HMMT25 と表記。

❌ B-5：SWE-bench Multilingual と TAU3-Bench の入れ替え

誤：「Qwen3.6-35B-A3B の SWE-bench Multilingual を 67.2 点」。

正：Hugging Face の公式モデルカードに記載されている値は、 67.2 は TAU3-Bench のスコア。 原稿は SWE-bench Multilingual のスコアと TAU3-Bench のスコアが入れ替わって記載されている。数値引用時は公式モデルカード最新版を直接確認のこと。

❌ B-6：RealWorldQA の Plus 側スコア

誤：「85.3 対 70.3 の大差」。

正：Qwen3.6-Plus の RealWorldQA 公式スコアは 85.4（85.3 ではない／0.1ポイントずれ）。 比較相手は Opus 4.5 の 77.0。

C. 価格・コスト表記の誤り

❌ C-1：国際リージョン価格と段階課金の欠落

誤：「入力 $0.29／1Mトークン、出力 $1.65／1Mトークン」。

正：Alibaba Cloud Model Studio の公表値はグローバル（0–256Kレンジ）で 入力 $0.276／1M、出力 $1.651／1M。 また、256K〜1M の長文コンテキスト帯では入力 $1.101／1M、出力 $6.602／1M と大幅に上がる段階課金である点が原稿で抜け落ちている。フルコンテキスト常用時には実効単価が4倍近くになる。

❌ C-2：Claude比は「17倍」ではなく「約18倍」

誤：「Claude Opus 4.6 に対して入力コストが約 17倍 安価」。

正：Opus 4.5 の入力単価 $5.00 に対し Qwen3.6-Plus の $0.276 なので、実際の倍率は 約18倍（TokenCost ほか複数ソースも "18x less" と明記）。 17倍は切り捨てが過ぎる。

❌ C-3：OAuth 無料枠は廃止済み

誤：「Qwen Code CLI 経由で OAuth ログインすれば1日1,000回の無料呼び出しが付与される」「恒常的な無料枠提供」。

正：Qwen Code 公式リポジトリ（QwenLM/qwen-code）のリリースノートに明記されている通り、 2026年4月13日に OAuth フリーティアは1日あたり1,000回から100回へ引き下げ、さらに2026年4月15日をもって OAuth 無料枠そのものが廃止されている。 本稿の想定時点である2026年4月20日においては「1日1,000回」は既に過去の数字であり、「恒常的な無料枠提供」という評価も完全に成立しない。少なくとも「短命な導入キャンペーン」と書くのが正。

D. アーキテクチャ・仕様の誤り

❌ D-1：「常時思考連鎖により切替を廃止」は不正確

誤：「すべてのプロンプトに対して強制的に思考連鎖を走らせる」意味での切替廃止。

正：公式ブログ仕様では、Qwen3.6-Plus にも依然として thinking / non-thinking の制御（preserve_thinking パラメータおよびモード指定）が残っている。 制御の抽象度が上がっただけで、ユーザー側からの思考制御インターフェース自体は撤廃されていない。

❌ D-2：Qwen3.5 ラインナップに「35B-A3B」は存在しない

誤：前世代として「Qwen3.5-35B-A3B」を挙げる。

正：Qwen3.5系列のオープンウェイトは Qwen3.5-397B-A17B（フラッグシップ）／Qwen3.5-122B-A10B／Qwen3.5-27Bが並んでおり、 Plus（クローズド）のバックボーンは 397B-A17B に相当。 「35B-A3B」サイズ帯は Qwen3.6 世代で初めて登場（2026年4月16日リリース）。

❌ D-3：35B-A3B のコンテキストは「262K 単独」ではない

誤：「262,144トークン」と単独で記載。

正：公式 Hugging Face モデルカードの仕様は "262,144 natively and extensible up to 1,010,000 tokens"（ネイティブ262K、YaRN等で約1Mまで拡張可）であり、拡張時1Mの上限記載が欠落している。

❌ D-4：Qwen3.5-Plus の旧コンテキストの扱い

誤：「Qwen3.5-Plus の 262K から一気に約4倍 へ拡張」。

正：先代 Qwen3.5-Plus はリリース当初からデフォルトで1Mコンテキストを備えていた。 ネイティブ262Kだったのはオープンウェイト版の Qwen3.5系（-27B 等）であり、ホステッドのPlusと混同している。Qwen3.6-Plus の1Mは「前世代 Plus と同じ1Mを維持しつつ、ネイティブ対応の幅を広げた」と書くのが正確。

❌ D-5：層数とパターンが Qwen3-Next と混同

誤：「48層、4層ごとに1層フル注意＋3層線形、1:3を12回繰り返し」。

正：これは Qwen3-Next 80B-A3B の仕様。 Qwen3.6-35B-A3B の公式 Hugging Face モデルカードに記載されている仕様は Number of Layers: 40、Pattern: 10 × (3 × DeltaNet → MoE + 1 × Attention) ＝ 4層ブロックを10回繰り返す40層構造。比率 3:1 は両世代共通だが総層数と繰り返し回数は取り違え。

❌ D-6：「12:1 が最もスパース」は過大評価

誤：「Qwen3.6-35B-A3B の12対1は、公開されている MoE の中で最も極端なスパース比率の一つ」。

正：同じアリババが2026年9月に公開した Qwen3-Next-80B-A3B は総80B／アクティブ約3Bで比率約27:1、 Kimi K2（総1T／アクティブ32B ≒ 31:1）、DeepSeek系の次世代モデルなど、12:1を上回るスパース比率のオープンMoEは複数存在する。「オープンウェイトMoEとしてはかなりスパースな部類」程度が妥当。

E. 日付・時系列の誤り

❌ E-1：Wukong 発表日は 4/2 ではなく 3/17

誤：「アリババは 2026年4月2日 に Wukong を正式発表した」（第11章・第14章）。

正：Wukongが正式発表されたのは 2026年3月17日（火）。 同日、アリババは組織再編（Alibaba Token Hub〈ATH〉新設）の翌日として、DingTalk連携のエンタープライズ向けAIエージェント基盤Wukongを招待制ベータで公開した。

補足：原稿が「4月2日」としたのは、Qwen3.6-Plus本体のOpenRouter公開日（2026年4月2日）との混同。両イベントは約2週間ずれており、Wukong はむしろ Qwen3.6-Plus リリースの半月前に先行発表された既存プラットフォームで、後から Qwen3.6-Plus を頭脳として統合した、というのが正しい時系列。

F. エコシステム・実績値の古さ

❌ F-1：Hugging Face 実績は「4億DL/14万派生」より大幅に大きい

誤：2026年4月時点の文脈で「累計 4億回ダウンロード／14万種の派生モデル」。

正：4億回ダウンロード・14万派生モデルは 2025年年央（7〜8月頃）時点のアリババ公式発表値。 2025年12月時点で単月ダウンロードが他8社合計を上回り、2026年初頭にはアリババ自身が「累計10億回ダウンロード超え」と発表している。したがって補遺篇が前提とする「2026年4月20日時点」の数値としては古く、実態より過少。より正確には 累計10億回超／派生モデル30万種以上 規模で記載すべき。

❌ F-2：量子化バリアントは10種ではなく20種超

誤：「Qwen3.6 公開から24時間以内に GGUF量子化版が10種類以上 出回った」。

正：2026年4月16日のリリース当日〜翌日にかけて、 Unsloth / Bartowski / mlx-community などから 20種類以上の量子化バリアント（GGUF / MLX / AWQ / FP8 等） が公開されたことが Hugging Face のリポジトリ履歴から確認できる。「10種類以上」は過少。

❌ F-3：Qwen3.5 シリーズの2026年2月リリース構成

誤：「Qwen3.5 シリーズとして 122B-A10B、35B-A3B、27B の3モデルが2026年2月24日に出た」と示唆（第14章・第17章関連）。

正：Qwen3.5 の2026年2月リリースのラインナップは実際には Qwen3.5-122B-A10B／Qwen3.5-Max／Qwen3.5-VL 系を中心としたファミリー。 「35B-A3B」は Qwen3.5 世代には存在せず、Qwen3.6 世代で初めて登場したサイズ帯（2026年4月16日リリース）。「Qwen3.5-35B-A3B」という表記は誤り。

G. その他比喩・スコアの過不足

❌ G-1：「100万トークン = 単行本10冊分」は英語換算

誤：プロローグで「100万トークン＝単行本約10冊分」。

正：日本語の場合、1トークン ≒ 0.6〜0.9文字相当で、100万トークンは日本語テキストで 概ね60〜90万字（単行本6〜9冊分）。 英語基準では約10冊相当。「単行本10冊分」は英語換算のみで、日本語読者向け表現としては過大表示。

❌ G-2：OpenRouter 記録は「1兆」ではなく「約1.4兆」

誤：「単一のAIモデルが24時間で 1兆トークン（1,000,000,000,000トークン） を処理するという記録」。

正：OpenRouter 公式アカウントおよび各種報道が明示した実際の数字は 約1.4兆トークン（約1,400,000,000,000） であり、「1兆の大台を史上初めて突破しつつ同日に1.4兆まで積み上がった」が正しい表現。公式記録の値を約0.4兆分過少に記述している。

❌ G-3：Wikipedia 6,000回という比喩は誇張

誤：「英語版Wikipediaの全記事を一日に 6,000回以上 読み込んだ計算に相当する」。

正：英語版Wikipedia全記事のトークン総量は、標準的な BPE トークナイザで概ね80億〜100億トークン程度。実際の記録値である1.4兆トークンで計算しても 約140回、 原稿の前提である1兆トークンでも約100回が上限で、「6,000回」は実際の40〜60倍に膨らませた誇張。「約100〜150回」ないし「英語版Wikipedia全体を1日で百数十回読み切れる規模」と書くのが妥当。
