PROS & CONS

⑧ メリット10選
/ デメリット5選

本気で Qwen3.6 を採用する前に読むべき「光と影」。26.5%の捏造率、日本語ベンチ未公開、OAuth 無料枠の廃止まで含めて率直に評価します。

CHAPTER 16
メリット総まとめ ― Qwen3.6 を選ぶ「10の理由」
  • 圧倒的なコスト効率：Qwen3.6-Plus は GPT-5.2 比で約 8〜9倍、Claude Opus 4.5 比で約 15〜18倍安価。エージェント用途のような大量トークン消費で月間コストが一桁落ちる。
  • 100万トークンの規格外コンテキスト：リポジトリ全体の読み込みや長文契約書の一括解析など、従来不可能だった「ワンショットでの全体把握」を可能にする。
  • Always-on CoT + preserve_thinking：マルチターンのエージェントタスクで思考の連続性が保たれ、従来の「AIが直前の会話を忘れる」現象が劇的に減る。
  • OpenAI / Anthropic 互換 API：既存コードを1行も変えずに Claude や GPT から乗り換えられる。
  • 主要CLI全対応：OpenClaw、Claude Code、Cline、Qwen Code、Kilo Code、OpenCode のすべてに即日対応。
  • Apache 2.0 オープンウェイト：35B-A3B は商用利用・再配布・ファインチューニング（追加学習）が完全に自由。企業が社内データで独自モデルを作ることも容易。
  • 民生機GPU・Apple Silicon対応：18〜20GB VRAM で動作するため、データを一切クラウドに送らず自社内で完結させたい金融・医療・官公庁用途にも適する。
  • マルチハードウェア最適化：AMD Instinct / NVIDIA全世代 / Apple Silicon / Intel など、異なるハード系統で公式対応が揃い、ベンダーロックインからの脱出が現実的。
  • 即戦力の実世界能力：視覚コーディング、GUIエージェント、ドキュメント解析（OmniDocBench 91.2点＝業界首位）など、エンタープライズ現場で即ROIを出せる能力群が標準装備。
  • 垂直統合された導入パス：Wukong や Qwen App といったエンタープライズ／コンシューマプロダクトへの垂直統合が進み、単体APIを使うだけでなくプラットフォーム全体としての導入経路が用意されている。
✅ メリットの総括
Qwen3.6 は「技術的に尖っているが使いにくい挑戦者」ではなく、 「エンタープライズ導入のあらゆる経路を整備しきった総合商社型のAI」として登場した、という理解が正しい。
CHAPTER 17
デメリットと影の部分 ― 26.5%の捏造率、自己報告ベンチ疑惑、日本語軽視

Qwen3.6 を冷静に評価するならば、その影の部分にこそ、これからの数ヶ月で開発者コミュニティが格闘することになる実務的論点が集中している。

① 26.5% という高い捏造率

⚠️ Fabrication Rate 26.5%
第三者ベンチマークサイト BridgeBench が公表した数値。AIにコード解析や事実関係の質問をしたとき、どの程度の頻度で根拠のない主張を作り出すかを測定したもの。 4回に1回はAPIの仕様や言語の挙動について「それっぽいが間違っている」回答を混ぜてくることを意味する。

同じベンチで Claude Opus や GPT-5.x は 10%台に収まっており、Qwen3.6 の信頼性は明らかにワンランク下。 Always-on CoT が思考を深める方向に機能する一方で、思考の中で生成された「もっともらしい中間仮説」がそのまま出力に混入しやすい、という構造的課題が指摘されている。これを補うには、ユーザー側が検索ツールやコード実行ツールを併用し、モデルの出力を外部的に検証するパイプラインを整えるしかない。

② 安全性スコアの 5ポイント遅れ

安全性スコアは 82.4（対 Claude Sonnet 4.5 の 87.2、GPT-5.4 Mini の 87.3）。 5ポイント差は絶対値としては小さいが、エンタープライズの監査（コンプライアンスチェック）では致命傷にもなり得る。

⚠️ 比較相手の訂正
原稿は「Claude（87.2）」「GPT-5.4（87.3）」を Opus 4.6 / GPT-5.4 本体のスコアのように書いていたが、 87.2 は Claude Sonnet 4.5、87.3 は GPT-5.4 Mini のスコア。 Opus 4.6 の BridgeBench 公開スコアは別数値（公表ベースでは 90 台半ば）。

加えて、中国発のモデルとして、特定の政治的・歴史的トピックに対するガードレール（応答制限）がどのように設計されているかの詳細は、公式ドキュメントからは十分に読み取れない。グローバル展開する企業が Qwen を全面採用する際には、自社でコンテンツフィルタリングを重ねる運用上の手間が発生する可能性が高い。

③ 自己報告ベンチの信頼性

アリババが公表するビジョン系ベンチマーク（特に RealWorldQA 85.4 など）について、自己報告である以上、外部検証を通過するまでは慎重に読むべき、という批判が独立系レビュアーから出ている。

⚠️ 訂正：RealWorldQA の比較相手
原稿は「Claude Sonnet 4.5（70.3）に 85.3 対 70.3 の大差を付けている」としていたが、 公式比較表で 70.3 と対比されているのは Gemma4-31B（オープンウェイト）であり、Sonnet 4.5 ではない。数値比較の文脈自体に誤りがあった。

④ 日本語ベンチマークの未公開

113言語対応と謳われながら、日本語特有のベンチマーク（JGLUE、JSQuAD、JCoLA など）での数値が公式には一切公開されていない。日本語の文学的・ビジネス的な微妙なニュアンスを扱う実務では、実測での検証が現時点では開発者自身の肩にかかっている。

⑤ Always-on CoT によるコスト上振れ

Always-on CoT によって出力トークンが膨らむケースが実運用で報告されており、preserve_thinking を正しく設定しないと、想定よりコストが上振れする事例もある。

⑥【追加】OAuth無料枠の消滅

⚠️ 訂正点：導入経済性の劣化
原稿の「OAuthで1日1,000回の恒常的な無料枠」は 2026年4月13日に100回/日へ縮小、4月15日に完全廃止されており、2026年4月20日時点では個人開発者・小規模スタートアップの導入ハードルが（相対的には依然低いものの）上がっている。

⑦【追加】256K〜1M帯の段階課金

原稿で欠落していた論点として、コンテキストが 256K を超えると 入力単価が $0.276 → $1.101、出力単価が $1.651 → $6.602 へと一気に約4倍へ跳ね上がる段階課金構造になっている。フルコンテキストを常用するユースケースでは、実効単価が公称の4倍近くになる点に要注意。

🌑 デメリットの総括
これらの弱点は、Qwen3.6 が「フロンティアに追いついた」という事実を否定するものではない。しかし「追いついた」と「抜いた」は別物であり、特に信頼性・安全性・多言語ニュアンスの三領域では、アリババがもう一段の努力を迫られる2026年下半期が待っている──というのが、公平な見立てだろう。

採用判断のチェックリスト

  • コード・エージェント用途で大量トークン消費 → Qwen3.6 は強力な候補。
  • 医療・金融・法務など「事実誤りが致命的」 → 外部検証パイプラインを必須で整備。
  • 日本語の微妙なニュアンスが重要 → 実測ベンチを自社で取るべき。
  • 256K以上のフルコンテキスト常用 → 段階課金を織り込んだコスト試算を。
  • オンプレ / プライバシー重視 → 35B-A3B オープンウェイトが現実解。
  • 中国リージョンの政治的・規制リスク → Omitted政治トピックのフィルタ自作を。
