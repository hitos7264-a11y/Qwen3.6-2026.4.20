🚨 最重要訂正ハイライト

原稿でとくに広範に拡散していた誤記TOP5。詳細は「ファクトチェック一覧」ページに全点掲載。

❌ 誤 → ✅ 正：比較対象のClaudeは「4.6」ではなく「4.5」
Qwen3.6-Plus 公式比較表の相手は Claude Opus 4.6 Claude Opus 4.5（80.9点）。 原稿が多用した「4.6」は第三者集計と混同した誤り。フロンティア対抗馬の筆頭も 4.5 が正。
❌ 誤 → ✅ 正：SWE-bench Pro は「大きく上回る」ではない
Opus 4.5 のSWE-bench Proは 45.9点 公式表では59.3点／TokenCost集計57.1点。 Qwen3.6-Plus の 56.6点は Opus 4.5 をやや下回る水準で、「大きく上回る」は誤り。
❌ 誤 → ✅ 正：RealWorldQA の比較相手取り違え
35B-A3B の 85.3（正確には85.4）と対比されているのは Claude Sonnet 4.5 (70.3) Gemma4-31B (70.3)。 Plus側のRealWorldQA比較相手は Claude Opus 4.5 の 77.0。Sonnet 4.5 の70.3は裏付け無し。
❌ 誤 → ✅ 正：価格は$0.29ではなく$0.276、かつ段階課金
グローバル(0–256K帯) 入力 $0.276 / 出力 $1.651。 256K–1M帯は 入力 $1.101 / 出力 $6.602 の段階課金に跳ね上がる点が原稿で欠落。対Opus 4.5 の倍率は 約18x（17xは切り捨て過ぎ）。
❌ 誤 → ✅ 正：AIME は「2026」ではなく「AIME25」
Qwen3.6-Plus の 95.3点は公式表記で AIME25（2025年のAIME）。 AIME 2026 のスコアは 35B-A3B 側のコミュニティ投稿値 92.7 として別途存在し、原稿は両者を取り違えている。HMMT25 は 96.7 で正。
❌ 誤 → ✅ 正：35B-A3B のコンテキストは「262K + 最大1M拡張可」
公式HF: "262,144 natively and extensible up to 1,010,000 tokens"。 原稿の「262,144トークン」単独表記は、YaRN 等での拡張上限約1Mの記載が欠落している。

📚 用語ミニ辞典

本サイト全体で繰り返し登場する専門用語の一行解説。

🔤 MoE（Mixture of Experts／混合専門家）
総パラメータ数が大きくても、推論時は「ルーター」が選んだ少数の専門家サブネットだけを動かす仕組み。Qwen3.6-35B-A3B は 35B のうち 3B だけを活性化する12:1のスパース設計。
🔤 コンテキストウィンドウ
AIが一度に読み取れるテキストの長さ＝「記憶容量」。100万トークンは日本語で概ね60〜90万字（単行本6〜9冊分）。英語換算ではおよそ10冊分。
🔤 CoT（Chain-of-Thought／思考連鎖）
AIが最終回答を出す前に内部で「なぜそう考えるか」のステップを書き下す手法。Qwen3.6-Plusは制御の抽象度を上げたが、thinking / non-thinking の制御（preserve_thinkingパラメータ等）は依然として利用可能。
🔤 MCP（Model Context Protocol）
AIが外部ツール・データソースと通信するための標準プロトコル。Qwen3.6-Plusはネイティブ対応で、MCPMarkなどの専用ベンチも用意されている。
🔤 SWE-bench
ソフトウェアエンジニアリングの実地試験。GitHub の実際のバグ修正イシューを解けるかを測る。Verified / Pro / Multilingual の3派生があり、エージェント性能の業界標準指標の一つ。
