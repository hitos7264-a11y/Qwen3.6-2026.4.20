1Mと3Bの衝撃

Qwen3.6-Plus の1Mコンテキストと思考保持機構／オープンウェイト版 35B-A3B の MoE 構造。「なぜ3Bアクティブで GPT-5.2 に迫れるのか」の技術的核心に踏み込みます。

CHAPTER 03
Qwen3.6-Plus の技術仕様 ― 1Mコンテキストと「思考保持」が再定義するエージェント基盤

スペック面から Qwen3.6-Plus を見ていくと、まず最初に目を奪われるのが、デフォルトで100万トークンに達するコンテキストウィンドウである。

⚠️ 訂正：「262Kから一気に約4倍」ではない
原稿は「Qwen3.5-Plus の 262K から一気に約4倍」としていたが、 先代 Qwen3.5-Plus はリリース当初からデフォルトで1Mコンテキストを備えていた。 ネイティブ262Kだったのはオープンウェイト版の Qwen3.5系（-27B 等）であり、ホステッドのPlusと混同している。 Qwen3.6-Plus の1Mは「前世代 Plus と同じ1Mを維持しつつ、ネイティブ対応の幅を広げた」と書くのが正確。

100万トークンが可能にするもの

100万トークンという枠は、中規模の GitHub リポジトリ全体、数百ページに及ぶ法務契約書束、数時間分の会議議事録と議事録音声テキストを一度にまとめてプロンプトに流し込んでも、なお余裕がある規模である。

  • 中規模リポジトリ全体を ワンショットで読み込み可能 ― チャンク分割RAGの複雑さが不要に。
  • 出力上限65,536トークンで、大規模設計書の一気書き、巨大ソースのリファクタ提案が一発完結。
  • 数時間分の会議議事録＋音声テキストをまとめて渡して要約・アクション抽出が可能。
  • 数百ページの法務契約書束をワンプロンプトで比較分析。
🔤 100万トークンは日本語で何冊分？
日本語の場合、1トークン ≒ 0.6〜0.9文字相当で、100万トークンは概ね60〜90万字。一般的な日本語単行本1冊が約10万字なので、6〜9冊分が妥当。 「単行本10冊分」は英語基準（1トークン ≒ 0.75ワード、単行本1冊 ≒ 75,000ワード）の換算であり、日本語読者向けとしてはやや過大表示。

「思考モード」の実像 ― 廃止ではなく抽象化

Qwen3.6-Plus は「Always-on Chain-of-Thought（常時思考連鎖）」をアーキテクチャ上の柱として据えている。

⚠️ 訂正：「切替を完全廃止」ではない
原稿は「thinking / non-thinking モード切替は完全に廃止された」と書いていたが、公式ブログ仕様では preserve_thinking パラメータ・モード指定は依然として残っている。 つまり「すべてのプロンプトに対して強制的に思考連鎖を走らせる」という意味での切替廃止ではなく、制御の抽象度が上がっただけで、ユーザー側からの思考制御インターフェース自体は撤廃されていない。

実装上のポイントは、モデル内部で「このタスクは軽いから思考を短く、重いから深く」という配分が自律的に行われている点だ。Qubridなど複数のベンチマーク環境での計測では、Qwen 3.6-Plus の平均応答時間はおよそ 13.9秒で、Qwen 3.5-Plus の 39.1秒 から劇的に改善された。旧世代で悪名高かった「延々と思考し続けて答えが出てこない」過剰思考問題が解消され、ベンチマークのconsistency指標では10.0点満点、flaky test（不安定なテスト失敗）件数ゼロという本番運用に耐える安定性が確認されている。

preserve_thinking という新パラメータ

エージェント開発者が最も注目すべきは、preserve_thinking という新たな API パラメータの導入だ。マルチターン会話の過去ターンで展開された「思考トレース」をすべて保持したまま次のターンに引き継げる仕組みで、従来のAPIでは思考内容がターンごとに捨てられていた課題を解決する。

原稿の「入力 $0.29／出力 $1.65」は$0.276 / $1.651 が正確な四捨五入。 さらに重大な欠落は、256K〜1Mの長文帯では入力 $1.101 / 出力 $6.602 に跳ね上がる段階課金という点である。 フルコンテキストを常用するユースケースでは、実効単価が4倍近くまで上がることに要注意。

CHAPTER 04
オープンウェイト版 Qwen3.6-35B-A3B の深層 ― 「3Bアクティブ」が切り開く新たな常識

Qwen3.6-35B-A3B の技術的ディテールに踏み込むと、そこには MoE アーキテクチャの成熟を物語る緻密な設計が姿を現す。総パラメータ 350億に対しアクティブは30億という、実効活性化率わずか8.6%という極端なスパース性が最大の特徴だ。

「小さな活性化量で大型モデルを凌駕」

Qwenチームが公式ブログで強調するのは、「この小さな活性化量で、Qwen3.5-27B や Gemma4-31B といったはるかに大型の稠密モデルをも凌駕する」というパフォーマンスの達成である。

原稿は「35B-A3B の 85.3 が Claude Sonnet 4.5 の 70.3 を上回る」としていたが、 公式 35B-A3B 比較表で RealWorldQA 70.3 点と対比されているのは Gemma4-31B（オープンウェイト）である。 クローズドモデル逆転エピソードとして語るのは、公式表の構造上は対象モデルの取り違えであり、Sonnet 4.5 とのRealWorldQAの70.3点という数値は裏付け無し。

ネイティブマルチモーダル＋切替可能な思考

Qwen3.6-35B-A3B はネイティブマルチモーダル機能を標準搭載しており、別途ビジョンエンコーダを外付けする必要がない。また「thinkingモード」と「non-thinkingモード」の両方を切り替えて使える仕様になっており、Plus版とは逆に、用途に応じて思考深度を制御できる柔軟性が残されている。

⚠️ 訂正：コンテキストは「262K（最大1M拡張可）」
原稿の「コンテキストウィンドウは 262,144 トークン」という単独記載には欠落がある。 公式 Hugging Face モデルカードの仕様は "262,144 natively and extensible up to 1,010,000 tokens" ＝ YaRN 等で約1Mまで拡張可能と明記されている。
