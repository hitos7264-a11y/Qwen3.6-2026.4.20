⚠️ 訂正：公式比較表のフロンティア4モデル
原稿が多用した「Claude Opus 4.6」「GPT-5.4」は公式比較表に含まれておらず、正しい比較軸は Claude Opus 4.5 / GPT-5.2 / Gemini-3 Pro / Kimi-K2.5。 本章以降のスコア比較は、すべてこの公式4モデルを基準として訂正しています。

35B-A3B も「オープンで衝撃」

オープンウェイト版の 35B-A3B に目を移すと、30億アクティブパラメータという極端に軽量な構成にもかかわらずSWE-bench Verified で 73.4点を叩き出している。RealWorldQA の 85.3 は、公式比較表では Gemma4-31B の 70.3 を大きく上回る数値として掲載されている（原稿の「Claude Sonnet 4.5」との対比は誤り）。

冷静な視点 ― 26.5%の捏造率

⚠️ 忘れてはいけない弱点
独立系第三者テスト（BridgeBench）では、Qwen3.6-Plus について「API仕様やプログラミング言語の挙動に関する主張で26.5%の高い捏造率」が計測されている。安全性ベンチの素点も 82.4点で、Anthropic・OpenAIが長年磨いてきた整列（alignment）の厚みに追いついていない。
⚠️ 訂正：安全性スコアの帰属
BridgeBench Security Bench v1 で 87.2 を記録しているのは Claude Sonnet 4.5（Opus 4.6 のスコアではない）。 同じく 87.3 は GPT-5.4 Mini（フルモデルではなくミニ版）。 Opus 4.6 の BridgeBench 公開スコアは公表ベースで 90 台半ばと別数値。

速度面の優位

OpenRouter 経由の実測で、Qwen3.6-Plus のスループットは 158トークン/秒前後。Claude Opus 4.5 のおよそ3倍の出力速度に相当する。先代 Qwen3.5-Plus の平均応答時間 39.1秒 から 13.9秒まで短縮された背景には、「過剰思考の矯正」と「効率的な推論トークン配分」の両方が効いている。

開発者体験 ― 乗り換えやすさの設計

  • OpenAI互換エンドポイント（dashscope-intl.aliyuncs.com/compatible-mode/v1）で既存SDKがそのまま動く。
  • Anthropic互換エンドポイントで Claude Code 用シェルスクリプトがそのまま走る。
  • MCPネイティブ対応・JSON構造化ツールコール・並列ツールコール・スキーマ適合が標準装備。
  • OpenClaw / Claude Code / Cline / Qwen Code 公式統合済み。
  • 「既存ワークフローを壊さずバックエンドだけQwenに差し替える」移行が可能。
⚠️ 訂正：恒常的な無料枠は2026/4/15に終了
原稿の「Qwen Code OAuth で1日1,000回の恒常的な無料枠」は 2026年4月13日に100回/日へ縮小、4月15日に廃止されている。 2026年4月20日時点では「短命な導入キャンペーン」として記述すべき。
