⑦ Wukongと
ハードウェア包囲網

3月17日に先行発表されたエンタープライズ基盤「悟空」。AMD Day 0、Apple Silicon、20種超の量子化バリアント ― Llama エコシステムを静かに侵食するハード対応の全景。

CHAPTER 14
Wukong（悟空）と「capability loop」 ― アリババのエンタープライズ覇権戦略
⚠️ 訂正：Wukong の発表日は「4月2日」ではなく「3月17日」
原稿は「アリババは 2026年4月2日に Wukong を正式発表した」と繰り返し記述していたが、 Wukong が正式発表されたのは 2026年3月17日（火）。 同日、アリババは組織再編（Alibaba Token Hub〈ATH〉新設）の翌日として、 DingTalk連携のエンタープライズ向けAIエージェント基盤Wukongを招待制ベータで公開した。

補足：ファイルが「4月2日」としたのは、Qwen3.6-Plus 本体の OpenRouter 公開日（2026年4月2日）との混同。 両イベントは約2週間ずれており、Wukong はむしろ Qwen3.6-Plus リリースの半月前に先行発表された既存プラットフォームであり、後から Qwen3.6-Plus を頭脳として統合した、というのが正しい時系列。

Wukong の設計思想

Wukong（悟空）は『西遊記』に登場する孫悟空に由来する、アリババのエンタープライズ業務自動化基盤である。単一のAIモデルを提供するのではなく、複数のAIエージェントをオーケストレーション（編成・協調動作）させることで、企業内の複雑な業務プロセス全体を自律化する。

対象業務従来Wukong
経理人手 + 複数SaaSエージェント群が分担
カスタマーサポートオペレーター中心Qwen3.6-Plus が頭脳
マーケティング複数ツール連携統一プラットフォーム
開発手動CI/CDGUIエージェント連携
在庫管理 / 棚卸し人手検査画像解析エージェント

capability loop（能力の循環）

Wukong の設計を貫く哲学が「capability loop」という概念である。原稿の用語でいえば「認識→推論→行動」の単一ワークフローに対応するが、アリババが強調するのは、このループが一回転で終わらず、行動の結果を再び認識として取り込み、次の推論・行動へと絶え間なく連鎖していく点だ。

💡 capability loopの実例（開発業務）
コードを書く →実行する → エラーを読み取る → 修正する → テストを通す → デプロイする
──このすべてを一つのループとして回し続ける仕組み。従来の「AIは一回出力したら終わり、人間が次を与える」という静的なパラダイムを破る。

Qwen3.6-Plus はこのループを単一モデルの内部で完結させられる初のフラッグシップとして位置づけられており、だからこそ「Real World Agent」という大仰な看板を掲げられる。また、アリババが「Qwen App」と呼ぶコンシューマー向けAIアプリケーションの背骨にもこの Wukong-Qwen3.6-Plus の組み合わせが配置されており、同社は自社の全AI体験を段階的に Qwen3.6 シリーズへ寄せていく方針を明確にしている。

CHAPTER 15
民生GPU・AMD・Apple Silicon ― ハードウェア包囲網の完成

Qwen 3.6-35B-A3B のハードウェア対応状況は、2026年4月時点ではるかに広範囲に広がっている。このハードウェア包囲網の広がりこそが、Meta が築いてきた Llama エコシステムの地位を Qwen が実質的に奪おうとしているプロセスの、最も目に見える部分である。

AMD による Day 0 サポートの意味

✅ Day 0 サポートとは
新モデルがリリースされたその日のうちに公式対応が発表されることを指す業界用語。 AMDは2026年4月16日のQwen3.6-35B-A3Bリリースと同時に、下記 GPU 全てでの動作検証を完了した。
AMD GPUHBM対応内容
Instinct MI300X192GBvLLM / SGLang
Instinct MI325X—ROCm 7.0 最適化
Instinct MI350X—同上
Instinct MI355X288GB HBM3EFP8量子化版公式提供

さらに、FP8（8ビット浮動小数点形式）量子化版「Qwen3.6-35B-A3B-FP8」を同時に提供し、EAGLE投機的デコーディング（AIが先回りして「次にこう答えるだろう」と予測することで生成速度を数倍に引き上げる最先端技術）にも対応した。これは、NVIDIA の H100 や H200 以外の選択肢を探している企業、あるいは米中摩擦下で先端GPUの輸出規制に苦しむ中国国内企業にとって福音となる発表である。

Apple Silicon 対応

コミュニティ発の Unsloth Q4_K_S GGUF を使うと、ファイルサイズは約20.9GBに収まる。

機器RAM実測スループット
MacBook Pro M3 Max64GB35〜50 tok/s
Mac Mini（最小構成）16GB動作報告あり
RTX 3090 / 409024GB75〜120 tok/s

「本格コーディングエージェントを、追加投資ゼロの手元のMacで24時間365日走らせられる」──少し前までは夢物語だった環境が、2026年4月に一気に現実化した。

ローカルランタイムの即時対応

llama.cpp、LM Studio、Ollamaといったローカル LLM ランタイムの側でも即座に対応が進んだ。

⚠️ 訂正：量子化バリアントは「20種類以上」
原稿の「リリース後24時間でコミュニティ量子化版が10種類以上出回った」は過少。 2026年4月16日のリリース当日〜翌日にかけて、 Unsloth / Bartowski / mlx-community などから 20種類以上の量子化バリアント（GGUF / MLX / AWQ / FP8 等） が公開されたことが Hugging Face のリポジトリ履歴から確認できる。

ハードウェア包囲網の全景

  • NVIDIA：H100/H200（FP16）、A100（INT8）、RTX 3090/4090（4bit）
  • AMD：Instinct MI300X/325X/350X/355X（Day 0対応、FP8版あり）
  • Apple Silicon：M1/M2/M3、MLXフォーマット公式提供
  • Intel：GPU/CPUでも準公式対応が進行中
  • ランタイム：vLLM / SGLang / llama.cpp / LM Studio / Ollama
  • 量子化：Unsloth / Bartowski / mlx-community ほか20種超
✅ 結論
異なるハードウェア系統で公式／準公式の最適化が揃っていることは、 ベンダーロックイン（特定企業の製品に縛られる状態）からの脱出を現実にする。 Meta の Llama が2023〜25年に築いた「オープンの基軸通貨」の座を、Qwen 3.6 はハードの側からも静かに侵食している。
